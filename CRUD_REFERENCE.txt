Reusable CRUD Reference (v2)
===============================

This project uses a config-driven CRUD structure. To add a new entity, copy an
existing CRUD config and wrapper, then change the entity-specific values. In most
cases you do not need to modify the shared page component.


Main files
----------

1. src/components/crud/CrudPage.js — shared list/create/edit/delete page
2. src/components/crud/CrudFormModal.js — shared form modal
3. src/components/crud/crudUtils.js — shared validation helpers
4. src/api/apiClient.js — Axios client with auth header and safe errors
5. src/config/routes.js — route, sidebar, and permission configuration
6. src/crud/<entity>CrudConfig.js — entity-specific table/form configuration
7. src/components/<entity>/List<Entity>.js — thin wrapper that passes the config to CrudPage


Recommended copy-paste workflow
--------------------------------

Use an existing module as the template:
- Employee CRUD: src/crud/employeeCrudConfig.js and src/components/employee/ListEmployee.js
- Role CRUD: src/crud/roleCrudConfig.js and src/components/role/ListRole.js

Example: create Category CRUD by copying Employee CRUD.


Step 1: Create the config file
------------------------------

Copy src/crud/employeeCrudConfig.js to src/crud/categoryCrudConfig.js and change:

- title: page heading shown on the screen
- entityName: singular label used in messages like "Create Category"
- apiEndpoint: base API URL, for example "/categories"
- permissions: permission names for view/create/update/delete
- tableColumns: columns shown in the table
- fields: form inputs for create/edit modal

Example shape:

```javascript
const categoryCrudConfig = {
    title: "Category Management",
    entityName: "Category",
    apiEndpoint: "/categories",
    idField: "id",
    permissions: {
        view: "categories.view",
        create: "categories.create",
        update: "categories.update",
        delete: "categories.delete",
    },
    tableColumns: [
        { data: "id", title: "ID" },
        { data: "name", title: "Name" },
    ],
    fields: [
        {
            name: "name",
            label: "Name",
            required: true,
        },
    ],
};

export default categoryCrudConfig;
```

Important notes:
- If the entity uses a dropdown, use a field with type "select" and set optionsEndpoint.
- If the entity needs a checkbox group, copy the pattern from roleCrudConfig.js.
- Make sure the field names match the backend request payload exactly.


Step 2: Create the wrapper component
------------------------------------

Create src/components/category/ListCategory.js:

```javascript
import React from "react";
import CrudPage from "../crud/CrudPage";
import categoryCrudConfig from "../../crud/categoryCrudConfig";

function ListCategory() {
    return <CrudPage config={categoryCrudConfig} />;
}

export default ListCategory;
```


Step 3: Register the route and sidebar entry
--------------------------------------------

In src/config/routes.js:

1. Import the new component:

```javascript
import ListCategory from "../components/category/ListCategory";
```

2. Add a new object to PROTECTED_ROUTES:

```javascript
{
    path: "/categories",
    permission: "categories.view",
    sidebar: {
        label: "Category Management",
        icon: "📁",
    },
    Component: ListCategory,
},
```

The router and sidebar will pick it up automatically.


Step 4: Make the backend API match
----------------------------------

The frontend expects the backend to support these endpoints:

- GET /api/categories
- POST /api/categories
- GET /api/categories/{id}
- PUT /api/categories/{id}
- DELETE /api/categories/{id}

Also make sure:
- The list endpoint returns a DataTables-compatible response.
- The create/update endpoints validate the incoming data.
- Permissions used in the config exist in the backend permission table. Role
  Management reads that table dynamically and automatically displays a checkbox
  group for each CRUD permission prefix (for example, categories.view,
  categories.create, categories.update, and categories.delete).
- If you use dropdowns, the options endpoint exists and returns data in a shape expected by the form.


What usually does not need changes
----------------------------------

You normally do not need to edit:
- src/components/crud/CrudPage.js
- src/components/crud/CrudFormModal.js
- src/components/crud/crudUtils.js

These are shared and already support standard CRUD behavior such as:
- search
- create/edit modal
- delete confirmation
- action buttons
- permission-based display

Only change them if you need a completely custom UI behavior.


Checklist
---------

1. Copy a CRUD config from src/crud/ and rename it for the new entity.
2. Update title, entityName, apiEndpoint, permissions, tableColumns, and fields.
3. Create a List<Entity>.js wrapper in src/components/<entity>/.
4. Import the component and add a protected route in src/config/routes.js.
5. Add matching Laravel API routes, controller/repository logic, validation, and permissions.
6. Run npm test and npm run build.
